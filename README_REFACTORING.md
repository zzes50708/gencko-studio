## ✨ 關鍵改進

### 1. 邏輯與數據分離 ✅
- **舊方案**：物種邏輯散布在 calcUtils.js 各處
- **新方案**：所有物種數據集中在 utils/genetics/ 配置文件

### 2. 可擴展性提升 ✅
- **舊方案**：新物種需修改計算引擎
- **新方案**：新物種無需修改任何計算邏輯，僅創建配置文件

### 3. 結果排序優化 ✅
- **舊方案**：按機率排序
- **新方案**：按「完整表現隱性基因數」優先，符合育種實務

### 4. 代碼品質提升 ✅
- 移除 100% 硬編碼物種檢查
- 統一配置物件結構
- 清晰的單向依賴關係

## ⚡ 關鍵特性

### 1. 物種配置物件結構

```javascript
{
    id: string,              // 物種 ID，如 'lg', 'aft'
    name: string,            // 物種名稱
    genes: Array,            // 基因定義 (可擴展)
    comboRules: Array,       // 品系組合規則 (可擴展)
    checks: Object,          // 物種檢查函數 (可擴展)
    warnings: Array          // 物種級警告 (可擴展)
}
```

### 2. 結果排序策略

```javascript
// 優先級：完整表現隱性基因數 > 機率
finalOutcomes.sort((a,b) => {
    if(a.completeExpressionCount !== b.completeExpressionCount) {
        return b.completeExpressionCount - a.completeExpressionCount
    }
    return b.prob - a.prob
})
```

### 3. 完整表現定義
