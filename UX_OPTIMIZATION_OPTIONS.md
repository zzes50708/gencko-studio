# 基因選擇 UX 優化方案 - 常見模式對比

用戶反饋：「下拉式選單沒了，每個基因都要搜尋太白吃了」

## 🎯 診斷

當前實現存在的問題：
1. ❌ 基因列表默認是**折疊狀態**（需要點擊類型才能展開）
2. ❌ 如果用戶不知道搜索功能，會覺得需要搜索才能看到基因
3. ❌ 移動端上基因列表隱藏在 modal 中，需要找到「新增基因」按鈕
4. ❌ 沒有快速查看「常用基因」的方式（每次都要打開下拉菜單）

---

## 📋 常見的基因選擇 UX 模式

### **方案 A：展開式列表（推薦 - 最傳統）**

**概念**：顯示全部基因類型，點擊類型展開該類型的基因列表

```
豹紋守宮基因選擇

[顯性 ▼]  ←─ 點擊展開/折疊
  ├─ 謎 (Mystery)
  ├─ 白黃 (White/Yellow)
  └─ 赤紅 (Crimson)

[隱性 ▼]
  ├─ 日蝕 (Eclipse)
  ├─ 暴風雪 (Blizzard)
  └─ ...

[共顯性 ▼]
  ├─ 雪花 (Snow)
  ├─ ...

[品系 ▼]
  ├─ 紅鑽石 (Red Diamond)
  └─ ...

[搜索框]  ↑─ 可選，用於快速過濾
搜索基因... (如:橘化、日蝕)
```

**優點**：
- ✅ 熟悉的操作方式（類似文件瀏覽器）
- ✅ 一次展開可以看到該類型的全部基因
- ✅ 清晰的層級結構
- ✅ 易於瀏覽，不需要搜索

**缺點**：
- ❌ 需要多次展開/折疊
- ❌ 51個基因分散在多個類型中
- ❌ 移動端上可能需要滾動

**適用場景**：
- 初級用戶（需要瀏覽所有選項）
- 不知道基因名稱的用戶

**實現方式**：當前實現 ✅（只需調整 UI，使基因列表默認展開）

---

### **方案 B：搜索優先 + 最近使用（推薦 - 混合模式）**

**概念**：搜索框在最上面，下面顯示「最近使用」的基因，然後才是分類列表

```
新增基因

┌─────────────────────────────┐
│ 🔍 搜索基因... (如:橘化、日蝕)  │  ←─ 默認焦點，可直接輸入
└─────────────────────────────┘

⭐ 最近使用 (Last Used)
  ├─ 橘化 (Tangerine)
  ├─ 日蝕 (Eclipse)
  └─ 雪花 (Snow)

顯性 (Dominant)
  ├─ 謎
  ├─ 白黃
  └─ ...

隱性 (Recessive)
  ├─ 日蝕
  ├─ 暴風雪
  └─ ...
```

**優點**：
- ✅ 常用基因快速訪問（最近使用記錄）
- ✅ 搜索是首選，但也支持瀏覽
- ✅ 兼顧新手和老手用戶
- ✅ 移動端友好（可保存到 localStorage）

**缺點**：
- ⚠️ 需要實現「最近使用」記錄存儲
- ⚠️ 初始沒有最近使用記錄（只在使用後才有）

**適用場景**：
- 經常進行重複配對（展會、批量配對）
- 需要快速切換多個親代

**實現難度**：⭐⭐☆ 中等（需要 localStorage）

---

### **方案 C：分頁式平鋪（簡潔 - 現代風格）**

**概念**：所有基因用 Tag/Chip 的形式平鋪展示，分頁或無限滾動

```
新增基因

🔍 搜索: [搜索框]

顯性  隱性  共顯性  品系  血系  [篩選 ▼]

┌──────────────────────────────────────┐
│ ✕ 謎  ✕ 白黃  ✕ 赤紅  ✕ 日蝕  ✕ 暴風雪 │
│ ✕ 雪花  ✕ 橘化  ✕ 血橘  ✕ 土匪  ✕ 蜜橘  │
│ ✕ 貝爾白化  ✕ 川普白化  ✕ 雨水白化  ...  │
└──────────────────────────────────────┘

[◄ 第1頁 ►]  或  自動加載更多...
```

**優點**：
- ✅ 一眼看到更多基因
- ✅ 現代化的 UI（材料設計風格）
- ✅ 支持多種篩選維度
- ✅ 移動端友好（響應式網格）

**缺點**：
- ❌ 篩選後可能還有很多結果
- ❌ 不如列表清晰（特別是有子分類時）
- ❌ 需要較寬的屏幕才能展示多個基因

**適用場景**：
- 寬屏幕（桌面端）
- 用戶熟悉現代 UI 風格

**實現難度**：⭐⭐⭐ 複雜（需要響應式網格、篩選邏輯）

---

### **方案 D：樹形結構 + 快速搜索（專業 - 企業級）**

**概念**：樹形導航，展示層級關係，結合搜索和標籤

```
新增基因

🔍 搜索框

[x] 按類型過濾:
  ☑ 顯性 (6)
  ☑ 隱性 (15)
  ☑ 共顯性 (8)
  ☑ 品系 (11)
  ☑ 血系 (5)

基因列表:
  顯性 Dominant (6)
    ├─ ☐ 謎 Mystery - ID: mystery
    ├─ ☐ 白黃 White/Yellow - ID: wyt
    └─ ☐ 赤紅 Crimson - ID: crimson
  
  隱性 Recessive (15)
    ├─ ☐ 日蝕 Eclipse - ID: eclipse
    │   [詳情] [歷史記錄]  ←─ 額外操作
    └─ ...
```

**優點**：
- ✅ 功能最完整
- ✅ 支持多維度篩選
- ✅ 清晰的層級和詳細信息
- ✅ 適合大型基因庫

**缺點**：
- ❌ 過度設計（對 51 個基因來說太複雜）
- ❌ 佔用空間大
- ❌ 移動端難以使用

**適用場景**：
- 未來擴展到 100+ 基因時
- 需要高級篩選功能

**實現難度**：⭐⭐⭐⭐ 很複雜

---

### **方案 E：Tab 切換 + 精簡列表（中等）**

**概念**：按基因類型用 Tab 切換，每個 Tab 只顯示該類型的基因

```
新增基因

[顯性] [隱性] [共顯性] [品系] [血系]
   ↑
  當前 Tab

🔍 搜索此類型的基因...

□ 謎 (Mystery)
□ 白黃 (White/Yellow)
□ 赤紅 (Crimson)
□ ...

[更多選項 ▼]
```

**優點**：
- ✅ 清晰的分類（每次只看一類）
- ✅ 減少列表長度（51個基因分成 5 類）
- ✅ 移動端友好（Tab 可滑動）
- ✅ 搜索範圍明確

**缺點**：
- ⚠️ 需要在 Tab 之間切換才能看到其他類型基因
- ⚠️ 跨類型搜索需要切換 Tab

**適用場景**：
- 中等規模基因庫
- 移動端優先設計

**實現難度**：⭐⭐☆ 中等

---

### **方案 F：Smart Dropdown（推薦 - 最實用）**

**概念**：結合「最近使用」+ 「搜索」+ 「快速訪問」，下拉菜單默認展開

```
新增基因

┌────────────────────────────────────┐
│ ⭐ 最近使用                          │ ←─ 已用過的基因（快速訪問）
├────────────────────────────────────┤
│ 📌 常用推薦（基於配對常見度）      │ ←─ 系統推薦的常用基因
├────────────────────────────────────┤
│ 🔍 搜索: [______________________]  │ ←─ 可按名稱搜索
├────────────────────────────────────┤
│ 顯性 (6)              [展開 ▼]      │ ←─ 可展開/折疊
│  ├─ 謎 (Mystery)      ☐            │
│  ├─ 白黃 (White/Yellow) ☐          │
│  └─ 赤紅 (Crimson)    ☐            │
│                                    │
│ 隱性 (15)              [展開 ▼]     │
│  ├─ 日蝕 (Eclipse)     ☐           │
│  ├─ 暴風雪 (Blizzard)  ☐           │
│  └─ ...                            │
│                                    │
│ 共顯性 (8)            [展開 ▼]     │
│ 品系 (11)             [展開 ▼]     │
│ 血系 (5)              [展開 ▼]     │
└────────────────────────────────────┘
  ✕ 關閉
```

**優點**：
- ✅ **結合所有優勢**（最近使用 + 推薦 + 搜索 + 瀏覽）
- ✅ 快速訪問最常用基因（減少操作步驟）
- ✅ 新用戶也能瀏覽全部基因
- ✅ 搜索是備選方案（不是必須）
- ✅ 移動端友好

**缺點**：
- ⚠️ 下拉菜單會較大（需要較多空間）
- ⚠️ 實現較複雜（需要存儲使用歷史）

**適用場景**：
- **最適合爬蟲繁殖者**（重複配對、常用基因）
- 移動端（展會環境）
- 桌面端

**實現難度**：⭐⭐⭐ 中等偏難

## 🎯 推薦方案

### **第一選擇：方案 F（Smart Dropdown）**

**為什麼最適合**：
1. ✅ **符合爬蟲繁殖者的真實需求**
   - 經常重複同樣的配對（常用基因）
   - 在展會上需要快速操作
   - 有些人知道基因名，有些人需要瀏覽

2. ✅ **兼顧所有用戶類型**
   - 新手：可以瀏覽全部基因
   - 老手：最近使用記錄加快操作
   - 搜索者：快速過濾

3. ✅ **移動端友好**
   - 展會場景優化
   - 單手操作（Tab + 輸入）

4. ✅ **實現可行性**
   - 基於當前代碼擴展
   - 只需添加 localStorage 存儲

### **第二選擇：方案 B（搜索 + 最近使用）**

如果覺得方案 F 過於複雜，方案 B 也很好，只是少了「推薦」部分。

### **第三選擇：方案 E（Tab 切換）**

如果想保持簡潔，Tab 切換也是不錯的折中方案。

## ❓ 問題診斷

**為什麼現在感覺「下拉式選單沒了」？**

可能的原因：
1. 🔴 **基因列表默認折疊**（需要點擊類型才能展開）
   - 用戶期望直接看到基因列表
   
2. 🔴 **搜索框提示不夠明顯**
   - 用戶沒注意到搜索框
   - 以為沒有其他選擇

3. 🔴 **沒有快速訪問常用基因的方式**
   - 即使能搜索，每次都要搜索也很煩

4. 🔴 **下拉菜單尺寸太大**（特別是移動端）
   - modal fixed 可能遮擋了視圖

**解決方案優先級**：
1. **🔴 立即修復**：讓基因列表默認展開（5分鐘）
2. **🟡 短期改進**：添加「最近使用」記錄（2小時）
3. **🟢 長期優化**：完整的 Smart Dropdown（4小時）
