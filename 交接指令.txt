【Gencko Studio 專案狀態交接與開發規範】
1. 架構說明 (Architecture & Tech Stack)
核心技術：Nuxt 3 (SSR/SSG) + Vue 3 (Composition API) + Pinia + Supabase。
重點功能：PWA (設定為 prompt 觸發無感熱更新)、全站 App 化行動端體驗（手機底部導航、手勢下拉燈箱、提示氣泡等）。
UI 與 CSS 系統：
採用 CSS 全域變數 控制日夜模式 (var(--card-bg), var(--txt), var(--bd), var(--pri))，結合 opacity 做視覺層次。
絕對禁止：寫死絕對色碼 (如 #fff, #333, #000)，以及嚴禁使用 :global(body.day-mode) 或 !important 進行樣式強制覆寫。所有顏色必須由變數系統自動接管。
排版原則：Mobile-First (手機優先)，大量運用 Grid 雙欄並排與 Flexbox，組件樣式需高度封裝在 <style scoped> 內。
效能考量：首屏圖片 (LCP) 使用原生 <img> 搭配 fetchpriority="high" 直連 CDN，非首屏維持 <NuxtImg loading="lazy"> 延遲載入。
2. 輸出規則 (Strict Output Rules)
單一檔案處理：一次僅處理「一個檔案」的建立或修改。完成後必須停下來，等待我回覆「下一個」或給予反饋後才能繼續。
禁止省略代碼：輸出程式碼片段時，絕對禁止使用「...」、「// 省略」等方式，必須給出該區塊（例如整個 <style scoped> 或完整 <template>）的完整程式碼，以便直接複製貼上。
精準定位：修改前必須簡短說明原因，再給出修改後的完整程式碼。
