# Gencko 官網體驗測試報告

> **測試日期**：2026-07-02
> **測試視角**：第一次造訪的潛在客人（桌機 1440px + 手機 375px）
> **測試方式**：本機 dev server（連線正式 Supabase 資料）逐頁走訪，實際點擊按鈕、觸發功能、檢查 DOM 與狀態
> **當下狀態**：展場模式為 **關閉**（價格正常顯示）；競標區目前**無進行中拍賣**

## 一、總體結論

整體是一個**功能豐富、信任感強、行動體驗完整**的利基電商網站。15 個主要頁面 SSR 全數正常（HTTP 200），核心購物、知識、工具、社群功能皆可運作。導航在桌機（下拉）與手機（底部 sheet）都已分成 3 個清楚群組。

**發現 1 個關鍵 bug（LINE 諮詢訊息亂碼）** 與數個中低優先度的體驗問題，詳見第四節。

**體驗評語**：新客人能順利找到在售個體、看懂賣點（100% 健康／基因正確／運輸賠償）、並透過 LINE 洽詢；但首頁 3D 動畫、以及「收藏後的 LINE 諮詢按鈕亂碼」是兩個最影響第一印象的點。

## 四、發現的問題（依優先度）

### 🔴 ① 關鍵：收藏清單的 LINE 諮詢按鈕訊息亂碼

- **位置**：`app.vue:400`（收藏數 > 0 時右下浮現的 LINE 諮詢按鈕）
- **現象**：預填訊息為亂碼 `Hi Gencko, ???閎閰Ｗ??嗉?皜銝剔?摰悅 (N?? ID嚗n<清單>`，客人點擊後 LINE 開啟會看到損毀文字。
- **成因**：殘留的 UTF-8 mojibake（先前 #11 亂碼清理漏掉此處）。
- **影響**：直接傷害「加入收藏 → 私訊洽詢」這條轉換路徑的第一印象。
- **修法**：把該字串改回正常中文，例如：`'Hi Gencko，我想諮詢收藏清單中的守宮（共 ' + N + ' 隻，ID：\n' + ...`。一行修正、低風險。

### 🟡 ② 周邊商品頁內容稀少

- `/merch` 目前僅少數項目/骨架。若非刻意，建議補齊商品或加「即將上架」說明；若無販售計畫，可考慮從導航降權。

### 🟡 ③ 競標區長期空狀態

- 目前無進行中拍賣。空狀態文案正確，但常態性空著會讓「線上競標」入口顯得冷清；可考慮無拍賣時在導航淡化，或空狀態加「下次開拍預告 / 通知我」。

### 🟡 ④ 首頁 3D 動畫的第一印象成本

- 首頁是全螢幕 3D 介紹動畫。雖已做 lazy 載入、reduced-motion 降載、且有「直接進入官網」出口與全域導航列，但新客人仍需多一步才到內容。可考慮首屏同時露出「看在售個體」CTA（先前評估的 #U1-B，屬產品決策）。

### 🟢 ⑤ 文章閱讀進度條未於載入時偵測到

- 文章內頁的閱讀進度條（navbar）未在初始載入時出現，可能為捲動觸發。建議人工確認捲動時是否正常顯示。

### 🟢 ⑥ 首頁存在兩個 h1（既有項）

- fallback 的 sr-only h1 與 3D 場景 h1 並存（疑 hydration）。非 WCAG 硬失敗，但屬 SEO 最佳實踐，建議日後釐清。

## 五、體驗亮點（正面）

- **信任訊號強**：每隻個體標示「100% 健康／基因正確／運輸賠償」+ 48 小時健康受理跑馬燈 + 特寵醫院查詢（78 間，附電話與導航）。
- **導航清晰**：桌機 3 群下拉、手機 3 群底部 sheet，資訊架構整齊。
- **購物輔助完整**：搜尋、篩選、排序、收藏、瀏覽紀錄、個體比較、相似推薦、已售「找相似」出口一應俱全。
- **知識深度**：文章庫（分類清楚）、基因圖鑑（23 條）、基因計算機（含說明 modal）、飼養/健康/評估互動工具。
- **社群導流**：LINE 社群（加入享優惠）+ 官方帳號 + IG/FB。
- **行動優先**：底部導航、瀏海安全區、翻卡、骨架載入、PWA。
- **無障礙**：全站 WCAG AA 對比、skip-to-content、表單 aria-label、3D canvas aria-hidden。
- **展場模式**：可由後台設定日期，展期自動隱藏價格。

---

## 六、建議優先順序

1. **立即修**：① LINE 收藏諮詢訊息亂碼（一行、影響轉換）。
2. **短期**：② 周邊商品補齊或降權、③ 競標空狀態優化。
3. **中期（產品決策）**：④ 首頁首屏並陳 CTA、⑤ 閱讀進度條確認、⑥ 雙 h1 釐清。
